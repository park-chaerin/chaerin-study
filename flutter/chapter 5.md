## 05-1 함수 선언과 호출하기

### 함수 선언 위치
- 다트에서 함수는 톱 레벨, 클래스의 멤버, 다른 함수 내에 선언 가능.
- 다트에서는 함수 오버로딩 제공 X 

#### 함수 오버로딩?
- 매개변수의 타입이나 개수를 다양하게 만들어서 이름이 같은 함수를 여러 개 만드는 방법
- 다트는 왜 오버로딩 지원을 안할까? '옵셔널 매개변수' 기능이 제공되기 때문.

### 매개변수 타입
- 함수의 매개변수는 타입을 명시하거나 var로 선언할 수도 있다. (타입 생략도 가능)
- var로 선언할 경우 함수의 매개변수 값이 대입되는 시점은 함수를 호출할 때이므로 dynamic 타입이 됨.
- 타입을 적지 않고 변수 이름만으로 선언해도 dynamic 타입.

### 함수의 반환 타입
- 함수를 반환할 때 타입 지정 가능.
- 반환할 데이터가 없으면 void로 선언.
- 반환 타입 데이터를 생략할 경우(void 함수인 경우) dynamic 타입이 됨.
- 반환 타입이 dynamic인 함수에서 return 문으로 데이터를 반환하지 않으면 자동으로 null값 반환.

### 화살표 함수
- 함수의 본문이 한 줄일 때, 중괄호로 묶지 않고 화살표 기호(=>)로 표현.
```dart
void printUser() => print("hello world");   //이런식으로
```

## 05-2 명명된 매개변수
- 보통 함수 호출 시 함수에 선언된 매개변수의 개수, 타입, 순서에 맞게 값을 전달해서 호출함. 
- 다트에서는 함수의 매개변수를 선택적으로 지정하는 '옵셔널'을 지원함.

#### 옵셔널 2가지 형태
1) 명명된 매개변수
2) 옵셔널 위치 매개변수

### 명명된 매개변수
- 함수를 호출할 때 데이터를 전달하지 않을 수 있고, 데이터 전달시 '이름: 값' 형태로 매개변수 이름과 값을 함께 전달.
- 명명된 매개변수 사용 시 함수 선언할 때 배개면수 타입과 이름을 중괄호로 묶어서 표현.
```dart
void some( {String? data1} ){
  print('data1:$data1');
  
  some(data1: 'world');   // '이름 : 값' 형태
}
```
### 명명된 매개변수 선언 규칙
- 명명된 매개면수는 중괄호 {}로 묶어서 선언.
- 여러 매개변수를 중괄호로 묶어 명명된 매개변수로 선언할 수 있음.
- 한 함수에서 명명된 매개변수는 한 번만 선언할 수 있고, 순서상 마지막에 선언해야 함.
- 명명된 매개변수에는 기본값을 설정할 수 있음.
```
void some1({String? data2, bool? data3},int data1){ }  // -> 오류!
// 명명된 매개변수는 맨 마지막 순서에 선언해야 함.

void some2(int data1, {String> data2, bool? data3}, {int? data4}){ }  // -> 오류!
//명명된 매개변수는 한 함수에 하나만 가능.

void some3(int data1, {String? data2, bool? data3}){ }  // -> 성공!
```

### 명명된 매개변수 호출 규칙
- 명명된 매개변수에 데이터를 전달하지 않을 수 있음.
- 명명된 매개변수에 데이터를 전달하려면 반드시 이름을 명시해야 함.
- 명명된 매개변수에 데이터를 전달할 때 선언된 순서와 맞추지 않아도 됨.
```
void some(int data1, {String? data2, bool? data3}){}    // 명명된 매개변수 선언 

some()      // -> 오류! 일반 매개변수 생략하면 안됨.
some(10)    // -> 성공! 일반 매개변수는 생략하지 않고, 명명된 매개변수는 생략 가능!
some(10, 'hello', true);  // -> 오류! 명명된 매개변수에는 이름과 값을 같이 전달해야함.
some(10, data2:'hello', data3:true);  // -> 성공! 
some(10, 'data:true', 'hello') // -> 성공! 명명된 매개변수 선언 순서는 섞여도 상관없음!
```

### 기본 인자 설정하기
- 명명된 매개변수에는 기본 인자 설정이 가능.
#### 기본인자란?
- 함수 호출 때 데이터를 전달받지 못하면 매개변수에 대입하는 기본값.
- 명명된 매개변수는 선택적이기 대문에 호출할 때 데이터를 전달하지 않을 수 있음.<br>
  (데이터 전달을 안하면 매개변수가 null이 되기 때문에 ?를 붙여서 null 허용으로 해주자.)

### 필수 매개변수 선언하기 : required
- 명명된 매개변수에서 반드시 값을 전달받도록 강제하는 예약어.
- required 사용 시 값 전달 필수!
```dart
someFun({required int arg1}){
  print('someFun()..arg1 : $arg1');
}

main() {
  someFun();    // 오류! required을 사용했으니 매개변수 필수로 해야함.
  someFun(arg1: 10);  //성공!
}
```

## 05-3 옵셔널 위치 매개변수

### 옵셔널 위치 매개변수
- 명명된 매개변수처럼 값은 선택이므로 전달받지 않을 수 있음.
- 명명된 매개변수와 달리 데이터 전달 순서를 맞춰서 호출해야 함. 

### 옵셔널 위치 매개변수 선언 규칙
- 매개변수들을 대괄호로 묶음.
- 함수의 마지막 매개변수에만 사용 가능함.
- 매개변수에 기본 인자를 설정할 수 있음.

### 옵셔널 위치 매개변수를 포함하는 함수 규칙
- 매개변수 이름은 생략
- 매개변수가 선언된 순서에 따라 값이 할당됨.
```
//옵셔널 위치 매개변수 선언
void some(int arg1, [String arg2 = 'hello', bool arg3 = false]) { } 

some();   // -> 오류! 일반 매개변수는 생략하면 안됨!
some(10);   // -> 성공! 옵셔널 위치 매개변수는 생략 가능.
some(10, arg2:'world', arg3:true);  // -> 오류! 명명된 매개변수와 달리 값만 전달해야함!
some(10, 'world', true);    // -> 성공! 옵셔널 위치 매개변수는 이름은 생략하고 값만 전달.
some(20, true, 'world');  // -> 오류! 옵셔널 위치 매개변수는 순서를 지켜서!
some(10, 'world')   // -> 성공! 옵셔널 위치 매개변수 값을 일부만 넣을 때는 타입을 주의하자.
some(10, true)  //-> 오류! 타입이 맞지 않으면 오류 발생!
```

## 05-4 함수 타입 인수
- 다트에서는 모든 데이터가 객체 -> 함수도 당연히 객체!
- 함수를 대입할 수 있는 객체 : 함수 타입. Function으로 선언. ( 매개변수로 함수 가능! ) <br>
  [Function 타입 예제 코드](Dart/Function.dart)
- 함수 타입 변수에 대입할 함수를 특정 형태로 한정하고 싶을 때는 
```dart
some(int f(int a)){ //-> 이와 같이 선언해주면 됨. int f(int a)
  f(30);
}
```

### 익명 함수
- 이름이 생략된 함수. 람다 함수라고 부름.
```dart
fun1(arg){        // -> 첫번째 함수
  return 10;
}

Function fun2 = (arg) {     // -> 두번째 함수. 매개변수와 함수의 내용은 있지만 이름이 없음.
  return 10;
};
```
- 익명함수는 이름이 없기 때문에 독자적으로 사용 X. Function 타입에 대입할 함수를 정의할 때 사용.

## 05-5 게터와 세터 함수

### 다트 언어에서의 get, set
- 데이터를 가져올 함수의 get 예약어 추가 시 함수를 변수처럼 이용 가능.
- 데이터를 변경할 함수에 set 예약어 추가 시 함수를 변수처럼 이용 가능.

get, set을 추가한 함수 예제
```dart
String _name = 'hello';

// get 예약어를 추가한 함수는 데이터를 가져오는 게터이기 때문에 매개변수 선언X.
String get name {    
  return _name.toUpperCase();
}
// set 예약어를 추가한 함수는 데이터를 변경하는 세터이기 때문에 매개변수 필요. 해당 매개변수로 데이터 변경.
set name(value){
  _name = value;
}
// 중요 ! get, set을 사용한 두 함수의 이름이 같음!
```
- 둘 중 하나만 선언도 가능함.
- 만약 get예약어로 게터만 선언 시, final 변수처럼 데이터를 가져오기만 할 뿐 바꿀 수 X

## 05-6 기타 연산자 알아보기
- 기초 연산자를 제외한 몇 가지 연산자를 알아보자.

### 나누기 연산자 : ~/

- 다트에서 나누기 연산자는 / , ~/ 가 존재.
- / 연산자 : 나누기 결과를 실수로 반환.
- ~/ 연산자 : 나누기 결과를 정수로 반환. (소수점 아래를 버림.)

### 타입 확인과 변환 : is, as
- is 연산자 : 타입을 확인해 true 혹인 false로 알려줌.
- as 연산자 : 타입을 바꿔줌.
```dart
class User{
  void some() {
    print("User ... some() ...");
  }
}
main() {
  Object obj = User();    //Object타입 변수 obj에 User 클래스 객체 생성.
  obj.some(); // -> 오류! Object클래스에는 some()메서드가 정의되어 있지 않음!
  
  //is 연산자를 사용해서 obj 객체가 User 타입인지 확인하고, 결과가 true면 자동으로 형변환.
  if (obj is User){
    obj.some();   //is 연산자를 사용해 형 변환을 했기 때문에 obj 객체는 User 타입이므로 오류X.
  }
  Object obj1 = User();
  (obj1 as User).some();  //
}
```

### 반복해서 접근하기 : .. , ?..
- nullable 객체일 때는 ?.. 사용.
- 아닐때는 .. 사용.
```
var user = User();
user.name = 'kkang'
user.age = 10;
user.some();

>>> 반복접근 연산자 사용시
User()
  ..name = 'kkang'
  ..age = 30
  ..some();
```

## 05-7 실행 흐름 제어하기

### for 반복문에서 in 연산자
```
for( var i = 0; i < List.length; i++)     //이렇게도 쓸 수 있지만
for(var x in list)  //이렇게도 사용 가능
```
- for 문에 범위 연산자(in)를 사용하여 간단하게 사용 가능.

### switch ~ case 선택문
- break : switch 문 나가기
- continue : 특정 위치(라벨)로 이동하기
- return : switch 문이 작성된 함수 종료하기(반환하기)
- throw : switch 문이 작성된 함수 종료하기(던지기)

### 예외 던지기와 예외 처리

#### 예외를 던지는 throw문
- 프로그램의 특정 시점에 예외를 발생시키고 싶을 때 사용.
```dart
some() {
  throw Exception('my exception');
}

// 이렇게 사용자가 만든 객체를 던져도 됨!
class User{}
some() {
  throw User();         
}
```

#### try~on~finally 예외 처리
- try문에서 작성한 코드에서 예외 발생 시 on 문이 실행되고, finally 문에는 예외와 상관없이 무조건 실행할 코드 작성.
